# 3D Drone Formation Simulation

Симуляция 3D формирования дронов, где главный дрон управляет дочерними дронами в сферической формации.

## Особенности

- Главный дрон в позиции (0, 0, 0)
- Настраиваемое количество дочерних дронов (по умолчанию 100)
- Сферическое формирование вокруг главного дрона
- Каждый дрон работает в отдельной горутине
- Плавное движение с сохранением формации
- Визуализация на OpenGL с управлением камерой
- Graceful shutdown через context
- Конфигурация через JSON файл или флаги командной строки
- Покрытие тестами

## Требования

- Go 1.21+
- OpenGL 4.1+
- GLFW 3.3+

## Конфигурация

### Файл окружения (.env)

При запуске приложение автоматически загружает конфигурацию из файла `.env` (если он существует):

```bash
# Запуск с конфигурацией из .env
go run cmd/app/main.go

# Указать другой путь к .env
go run cmd/app/main.go -env=.env.production
```

Пример `.env`:

```env
# Количество дочерних дронов
DRONE_COUNT=100

# Минимальная дистанция от главного дрона
MIN_DISTANCE=5.0

# Максимальная дистанция от главного дрона
MAX_DISTANCE=10.0

# Интервал обновления позиции (в наносекундах)
UPDATE_INTERVAL=2000000000

# Радиус формирования
FORMATION_RADIUS=7.5

# Вариация движения для реалистичности (0-1)
MOVEMENT_VARIATION=0.5

# Коэффициент сглаживания движения (0-1)
SMOOTHING_FACTOR=0.2

# Режим отладки (true/false)
DEBUG=false

# Включить визуализацию (true/false)
VISUALIZATION_ENABLED=true

# Ширина окна визуализации
WINDOW_WIDTH=800

# Высота окна визуализации
WINDOW_HEIGHT=600
```

### Файл конфигурации (JSON)

Сгенерируйте файл конфигурации:

```bash
make gen-config
```

Параметры конфигурации:

```json
{
  "drone_count": 100,
  "min_distance": 5.0,
  "max_distance": 10.0,
  "update_interval": 2000000000,
  "formation_radius": 7.5,
  "movement_variation": 0.5,
  "smoothing_factor": 0.2,
  "debug": false,
  "visualization_enabled": true,
  "window_width": 800,
  "window_height": 600
}
```

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `drone_count` | Количество дочерних дронов | 100 |
| `min_distance` | Минимальная дистанция от главного дрона | 5.0 |
| `max_distance` | Максимальная дистанция от главного дрона | 10.0 |
| `update_interval` | Интервал обновления позиции (нс) | 2s |
| `movement_variation` | Вариация движения для реалистичности | 0.5 |
| `smoothing_factor` | Коэффициент сглаживания (0-1) | 0.2 |
| `debug` | Режим отладки | false |
| `visualization_enabled` | Включить визуализацию | true |
| `window_width` | Ширина окна визуализации | 800 |
| `window_height` | Высота окна визуализации | 600 |

> **Приоритет конфигурации:** Флаги командной строки > `.env` файл > JSON файл > значения по умолчанию

## Как это работает

1. Главный дрон инициализируется в позиции (0, 0, 0)
2. Создаётся заданное количество дочерних дронов
3. Каждый дочерний дрон размещается в сферической формации
4. Дроны обновляют позиции в отдельных горутинах
5. Визуализация отображает дроны в реальном времени
6. При получении сигнала (Ctrl+C) происходит graceful shutdown

## Расширение

### Добавление нового поведения дронов

1. Создайте новый контроллер в `internal/services/`
2. Реализуйте интерфейс поведения
3. Подключите в `SimulationService`

### Добавление новой визуализации

1. Реализуйте интерфейс `visualization.Visualizer`
2. Создайте фабрику для визуализатора
3. Подключите в `visualization.Manager`
